Day 3: Debugging and Reverse Engineering

Challenge instructions

For this lesson, you'll need the following two tools:

  • Ghidra, a tool made by the NSA for reverse engineering software on tons of architectures

    • You need java to run this! Follow the installation instructions on the Ghidra page!

  • The GNU debugger, a tool made by GNU project for debugging, commonly referred to as GDB.

    • Linux: Install with your package manager.

    • Windows: Install MSYS2 and use mingw64. Run with "pacman -S mingw-w64-x86_64-gdb".

    • MacOS: use brew (untested).

For each challenge, you will download a file and either inspect it with Ghidra or load it in to QEMU and debug it with GDB. Each flag begins with "CTF-" and ends with ";".

You may want to start qemu with the flags "-S" and "-s".

Run gdb and connect to qemu with "target remote localhost:1234".

Challenges 1-3: GDB

Challenge 1: String on the Heap

To play this challenge, download the following file: os-image-ctf1.bin

The heap in our kitty kernel starts at 0x10000. Can we read that memory?

Challenge 2: Calling a function

To play this challenge, download the following file: boot_sector-ctf2.bin

Our boot sector is loaded at 0x7c00, and in the third sector there exists a function to load in the second sector and print a string located there. Can we call that function?

Challenge 3: Inspect the registers

To play this challenge, download the following file: boot_sector-ctf3.bin

Instructions are in the binary :)
